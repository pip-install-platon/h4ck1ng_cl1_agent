# Общая карта методологии (промпт / будущий GraphRAG)

Язык ниже — рабочий чеклист (англ. термины как в индустрии). Размер в промпте ограничен **`COPILOT_TACTICS_MAX`**; свой файл: **`COPILOT_KNOWLEDGE_PATH`**.

### OWASP WSTG (веб)

Структурированное тестирование веб-приложений и API — по **[OWASP Web Security Testing Guide — `document/`](https://github.com/OWASP/wstg/tree/master/document)**. Там же оглавление по фазам SDLC (раздел 3) и чеклисты **4.x**: разведка (4.1), конфигурация и деплой (4.2), identity / authentication / authorization (4.3–4.5), сессии (4.6), валидация ввода и инъекции (4.7), ошибки и криптография (4.8–4.9), бизнес-логика (4.10), клиент (4.11), API включая GraphQL (4.12), отчётность (5). Сверяй план и нахождения с WSTG, не выдумывай отсутствующие категории тестов.

### OWASP MASVS (mobile)

Базовые требования и модель угроз для мобильных приложений (iOS/Android): **[OWASP MASVS — репозиторий](https://github.com/OWASP/masvs/)** (`Document/`, `controls/`, `OWASP_MASVS.yaml`). В экосистеме OWASP MAS цепочка **MASVS → MASWE** (классификация слабостей) **→ MASTG** (руководство по тестированию). Для мобильного скоупа сверяй проверки с MASVS/MASTG, не подменяй их абстрактным веб-чеклистом.

## Information Gathering

### Passive reconnaissance

- WHOIS, регистрационные данные домена.
- DNS-анализ, перечисление субдоменов.
- Пассивный сбор (Shodan, Censys и аналоги).
- OSINT: соцсети, форумы, утечки; вакансии (стек и системы); репозитории кода (GitHub и др.) на секреты и артефакты.
- SSL/TLS: издатель, сроки, цепочка.
- Google dorking на чувствительные находки.

### Active reconnaissance

- Сканирование сети и приложений (Nmap, Nessus и т.п.).
- Полное перечисление субдоменов.
- Обход веба на скрытое/динамическое содержимое.
- Топология сети, устройства.
- Стек: технологии, платформы, фреймворки.
- Типовые слабости: дефолтные учётки, непропатченные системы.
- Утечки: метаданные, HTML-комментарии и т.д.

### Social engineering (векторы разведки)

- Публичные каналы организации / сотрудников.
- Оргструктура и персоналии из открытых источников.

## Vulnerability analysis

### Automated scanning

- Валидация и приоритизация выдачи сканеров.
- Проверка известных CVE/эксплойтов под версии **по фактам**.
- Инструменты оценки уязвимостей — как входные данные, не как истина.

### Manual testing

- Веб: SQLi, XSS, CSRF, IDOR, небезопасная десериализация.
- Fuzzing новых векторов.
- Конфигурации серверов и приложений.
- Код-ревью где доступно.
- Аутентификация и авторизация.
- Утечки данных (URL, API).
- Сессии: fixation и смежные ошибки.

### Network and infrastructure

- Misconfig сетевого железа (firewall, router).
- Криптография и TLS на сервисах.
- API: аутентификация, авторизация, избыточные данные.
- Логирование и мониторинг.
- Сторонние компоненты и библиотеки.

### Compliance mapping (по запросу заказчика)

- OWASP Top 10, NIST и др. как **рамки классификации**, не замена фактам.
- Привязка находок к внутренним политикам и регламентам (PCI DSS, ISO 27001) — если это в скоупе отчёта.

## Exploitation

### Initial access

- Фишинг как вектор (если в скоупе).
- Эксплуатация известных уязвимостей подтверждённых версий.
- Слабые/дефолтные учётные данные.
- Фреймворки эксплуатации (например Metasploit) — сценарии под цель.

### Privilege escalation

- Локальный и вертикальный рост привилегий на захваченном хосте.
- Ошибки бизнес-логики приложений.

### Lateral movement

- Движение по сети, доступ к соседним системам и ресурсам.

### Security evasion

- Обход WAF, 2FA и аналогичных контролей (если в скоупе).
- Обход/стабильность против AV, IDS/IPS.
- Нестандартные и 0-day цепочки — только с явным указанием в скоупе и версиях.

### Documentation

- Шаги воспроизводимы, логирование действий для отчёта и повтора.

## Post-exploitation

### Impact analysis

- Критичные хранилища данных.
- Бизнес- и техвлияние находок.
- Реалистичность эксплуатации вне лаборатории.

### Persistence and cleanup

- Закрепление — только если это описано в скоупе.
- Удаление своих артефактов, тестовых учёток, лишнего ПО.
- Возврат к согласованному состоянию; проверка отсутствия «хвостов».

### Data handling

- Работа с чувствительными данными по правилам заказчика.
- Поиск учёток и секретов в памяти/конфигах.
- Моделирование эксфильтрации — если оговорено в скоупе.

### Documentation

- Полная фиксация изменений в системах.

## Specialized testing considerations

### IoT device testing

- Анализ прошивок на уязвимости.
- Безопасность протоколов связи (MQTT, CoAP и др.).
- Аппаратная часть: JTAG, UART и смежные интерфейсы.
- Безопасность OTA-обновлений.
- Дефолтные конфигурации и захардкоженные учётные данные.
- RF: Bluetooth, Zigbee и другие радиоканалы.
- Физические контроли доступа к устройствам.

### Container security

- Конфигурации Docker (daemon, namespaces, capabilities, образы).
- Безопасность Kubernetes-кластера (RBAC, etcd, network policies, admission).
- Сканирование образов контейнеров на CVE и misconfig.
- Runtime-мониторинг (поведение workload в рантайме).
- Service mesh: политики трафика, mTLS, секреты.
- Оркестрация: kube-apiserver, компоненты control plane, workload isolation.
- Защита container registry (доступ, подпись образов, уязвимые базовые слои).

### CI/CD pipeline security

- SCM (Git): ветки, хуки, защита от подмены пайплайна.
- Безопасность сборки (агенты, кэш, зависимости, supply chain).
- Репозитории артефактов (реестры пакетов, права, retention).
- Деплой: окружения, откат, подпись релизов.
- IaC (Terraform, Helm и др.): статический анализ, drift, секреты в коде.
- Управление секретами в пайплайне (не в логах, не в коде).
- Разграничение доступа к пайплайну и секретам.

### Cloud infrastructure

- Обзор конфигураций (CSPM-подход): типичные ошибки в облаке.
- IAM: избыточные права, trust relationships, ключи и роли.
- Хранилища: S3 / Blob — ACL, публичность, шифрование, версионирование.
- Сеть: security groups, NACLs, firewall rules, exposure сервисов.
- Serverless: функции, триггеры, переменные окружения, доступ к данным.
- Типовые misconfig и облако-специфичные цепочки (метаданные инстансов, SSRF к metadata и т.п.).
