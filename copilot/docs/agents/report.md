# Report-Agent — отчётность (`module: report`)

## Задача

Собрать из **графа текущего engagement** воспроизводимую историю: timeline шагов, **CVE ↔ находка ↔ тактика** (ATT&CK где применимо), использованные инструменты и модули (nmap, MSF, nuclei…), рекомендации **remediation**. Подготовить запись для **Report Library** (`library/reports/`) для подсказок будущим прогонам.

## Источники правды

- Локальный граф (узлы/рёбра, evidence).
- Метаданные CVE: **по необходимости fetch из NVD** в артефакт прогона (кэш), без подмены официального источника.
- Exploit-DB ID / имена MSF-модулей — из полей графа, заполненных Vuln/Exploit.

## Выход

- `graphOps`: узел `engagement_report`.
- **JSON** в библиотеку отчётов; позже **Markdown** для людей (тот же контент, два представления).

## Границы

Не выполняет новые атаки и не меняет приоритеты — только **документирует** сделанное.
