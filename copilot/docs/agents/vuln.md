# Vuln-Agent — анализ уязвимостей (`module: vuln`)

## Задача

Сопоставить наблюдения IG/сканеров (версии, баннеры, вывод nuclei, конфиги) с **известными уязвимостями**: CVE, CWE, кандидатные записи **Exploit-DB**, при необходимости привязка к техникам **MITRE ATT&CK** (discovery, initial access patterns — как **классификация**, не замена тактического плана).

## Внешние данные: не своя БД

- **NVD** (API/feeds) — метаданные CVE, CVSS, описание.
- **Exploit-DB** — `searchsploit`, при необходимости синхронизация индексов для **офлайн-поиска** (данные Exploit-DB, а не выдуманная схема в продукте).
- **OSV** — для цепочек зависимостей (опционально).
- **Срезы MITRE** — загрузка справочных JSON/файлов под задачу (тактика ↔ техника) для навешения тегов на находки.

Модуль делает **корреляцию и обоснование**; долговременное хранение — **граф engagement** + кэш артефактов запросов, без конкурирования с NVD как с «главным источником».

## Инструменты на Kali

- **Metasploit** — **auxiliary** / **scanner** / проверочные модули (`search type:auxiliary`, `type:scanner`). Переход к `exploit/` — только если Team-Lead выдал отдельную задачу **Exploit** или явно разрешил в `directive`.
- **nuclei** — CVE/экспозиция шаблоны по согласованным тегам.
- **searchsploit** — сопоставление продукта/верс с локальным индексом Exploit-DB.
- **nikto**, **wapiti** — по skillPack и типу цели.

## Выход

Узлы `vulnerability` / `finding` с полями вроде `candidate_cves[]`, `cwe`, `confidence`, `source` (nvd, nuclei, msf), рёбра к `service` / `host`. Каждая смысловая строка в `outputFindings` с evidence.

## Границы

**Приоритизация** и решение «проверять эксплуатацию сейчас» — **Team-Lead**. Vuln **не** выполняет полный post-exploitation.
